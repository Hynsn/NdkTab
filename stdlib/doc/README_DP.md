## 设计模式

设计模式是针对常见问题的通用解决方案。

比如保证一个类只有一个实例 -> 单例

搭建两个不兼容接口之间的桥梁 ->适配器

为Library、framework提供一个简洁的接口 ->外观

设计模式的必要性

没有设计模式：代码维护成本高，改进空间小

优点

- 减少垃圾代码，提高代码可维护性和可扩展性

- 减少沟通成本

- 开拓思维，提升解决问题的能力
